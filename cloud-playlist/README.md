# Облачный плейлист

Эта папка — «облако» для плеера: файлы отсюда раздаются через бесплатный CDN
[jsDelivr](https://www.jsdelivr.com/) поверх этого GitHub-репозитория. Плеер
сам подтягивает `manifest.json` при запуске и по кнопке «Обновить облачный
плейлист» — переустанавливать приложение не нужно, обновление видят все
пользователи сразу после публикации.

## Как добавить треки

Просто положите аудиофайлы (`.mp3`, `.m4a`, `.wav`, `.flac`, `.aac`, `.ogg`) в
`cloud-playlist/tracks/` и запушьте изменения — например, через веб-интерфейс
GitHub: откройте папку `cloud-playlist/tracks/` → **Add file → Upload files**
→ перетащите файлы → **Commit changes**. Это работает и с телефона.

**`manifest.json` вручную редактировать не нужно** — после пуша
автоматически запускается GitHub Action
(`.github/workflows/cloud-playlist.yml`), который:
1. пересобирает `manifest.json` по факту содержимого `tracks/`;
2. читает название и исполнителя прямо из ID3-тегов файла (если тегов нет —
   берёт имя файла);
3. коммитит обновлённый `manifest.json` обратно в репозиторий;
4. сбрасывает кэш jsDelivr, чтобы обновление долетело до плеера сразу, а не
   через несколько часов.

Через 1-2 минуты после загрузки файла новый трек уже доступен всем
пользователям плеера по кнопке «Обновить облачный плейлист» (или при
следующем открытии приложения).

## Как удалить трек

Удалите файл из `cloud-playlist/tracks/` и запушьте изменения — Action сам
уберёт его из `manifest.json`, и при следующей синхронизации плеер уберёт
трек из облачного плейлиста у всех пользователей.

## Если теги файла отсутствуют или неверные

Название трека возьмётся из имени файла (`_`/`-` заменяются на пробелы), а
исполнитель останется пустым. Проще всего поправить это через теги самого
mp3-файла (любой тег-редактор, например Mp3tag/Kid3, или `ffmpeg -metadata
title="..." -metadata artist="..." -codec copy in.mp3 out.mp3`) и перезалить
файл — Action подхватит теги при следующем пуше.

## Запуск вручную (локально)

```bash
npm run generate:cloud-manifest
```

Пересоберёт `cloud-playlist/manifest.json` по текущему содержимому
`cloud-playlist/tracks/` — полезно, чтобы проверить результат перед пушем.

## Формат manifest.json (генерируется автоматически)

```json
{
  "playlistName": "Название облачного плейлиста",
  "playlistCover": "https://ссылка-на-обложку-плейлиста.jpg",
  "tracks": [
    {
      "file": "tracks/song.mp3",
      "title": "Song Title",
      "artist": "Artist Name"
    }
  ]
}
```

`playlistName` и `playlistCover` при регенерации сохраняются как есть — их
можно один раз поправить вручную в `manifest.json`, они не перезатираются
автоматикой (перезаписывается только массив `tracks`).

Плейлист создаётся в приложении автоматически при первой синхронизации —
отдельно ничего создавать не нужно. Локально загруженные в этот же плейлист
файлы синхронизация не трогает.
